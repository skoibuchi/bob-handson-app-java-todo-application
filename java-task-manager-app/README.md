# TaskManager

シンプルなタスク管理Webアプリケーション

## 概要

TaskManagerは、個人のタスク管理を支援するシンプルなWebアプリケーションです。
タスクの作成、表示、更新、削除（CRUD操作）を行うことができます。

## 主な機能

- **タスク一覧表示（カンバンボード形式）**: 全タスクをステータス別に4列のカンバンボード形式で表示
- **タスク詳細表示**: 個別タスクの詳細情報を表示
- **タスク作成**: 新規タスクの作成
- **タスク編集**: 既存タスクの編集（一覧画面から直接アクセス可能）
- **タスク削除**: 不要なタスクの削除
- **ステータス管理**: 未着手、進行中、保留、完了の4つのステータス
- **優先度管理**: 高、中、低の3段階の優先度
- **期限設定**: タスクの期限を設定可能
- **レスポンシブデザイン**: デスクトップ・モバイル対応

## セットアップ手順

### 1. リポジトリのクローン

```bash
git clone <repository-url>
cd java-task-manager-app
```

### 2. アプリケーションのビルド

```bash
mvn clean install
```

### 3. アプリケーションの起動

```bash
mvn spring-boot:run
```

または、JARファイルを直接実行：

```bash
java -jar target/task-manager-1.0.0.jar
```

### 4. ブラウザでアクセス

```
http://localhost:8080
```

アプリケーションが起動すると、自動的にタスク一覧画面が表示されます。

## H2 Consoleへのアクセス

開発中にデータベースの内容を確認したい場合は、H2 Consoleにアクセスできます。

```
URL: http://localhost:8080/h2-console
JDBC URL: jdbc:h2:mem:taskdb
Username: sa
Password: (空欄)
```

## データの永続化

このアプリケーションは**ファイルベースのH2データベース**を使用しており、データは永続化されます。

- **データファイル**: `./data/taskdb.mv.db`
- **動作**: アプリケーションを再起動してもデータが保持されます

### データのリセット

初期状態に戻したい場合は、以下のコマンドを実行してください：

```bash
# dataフォルダを削除
rm -rf data

# アプリケーションを再起動
mvn spring-boot:run
```

### 初期データについて

初回起動時のみ、`data.sql`から以下のサンプルデータが投入されます：

1. Spring Bootの学習（進行中、優先度：高）
2. データベース設計（完了、優先度：高）
3. 画面デザイン（未着手、優先度：中）
4. テスト実施（保留、優先度：中）
5. ドキュメント作成（未着手、優先度：低）

**注意**: 2回目以降の起動では、既存データが保持されるため初期データは投入されません。

## 使い方

### タスクの作成

1. タスク一覧画面の「新規作成」ボタンをクリック
2. タイトル、説明、ステータス、優先度、期限を入力
3. 「保存」ボタンをクリック

### タスクの表示

- **一覧表示（カンバンボード）**: トップページで全タスクをステータス別に4列で表示
  - 未着手（TODO）
  - 進行中（IN_PROGRESS）
  - 保留（ON_HOLD）
  - 完了（COMPLETED）
- **詳細表示**: タスクカードの「詳細」ボタンをクリック

### タスクの編集

方法1: 一覧画面から直接編集
1. タスクカードの「編集」ボタン（鉛筆アイコン）をクリック
2. 情報を編集
3. 「保存」ボタンをクリック

方法2: 詳細画面から編集
1. タスク詳細画面の「編集」ボタンをクリック
2. 情報を編集
3. 「保存」ボタンをクリック

### タスクの削除

1. タスク詳細画面の「削除」ボタンをクリック
2. 確認ダイアログで「OK」をクリック

## 注意事項

- **データの永続化**: H2データベースをファイルベースで使用しており、データは永続化されます
- **データファイル**: `./data/taskdb.mv.db`に保存されます
- **単一ユーザー**: 認証機能はなく、単一ユーザーでの利用を想定しています
- **デモ用途**: 本アプリケーションはデモ用途として作成されており、本番環境での使用は想定していません

## トラブルシューティング

### ポート8080が既に使用されている

`application.properties`でポート番号を変更してください：

```properties
server.port=8081
```

### アプリケーションが起動しない

- Java 17以上がインストールされているか確認してください
  ```bash
  java -version
  ```
- `mvn clean install`を実行して依存関係を再インストールしてください
